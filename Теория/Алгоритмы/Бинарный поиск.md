#flashcards 
***
Алгоритм, на вход которому подается отсортированный список элементов. Если элемент, который мы ищем, присутствует в списке, то бинарный поиск возвращает ту позицию, в которой он был найден. В противном случае он возвращает `null`.
***
***Принцип работы.***
Пусть есть отсортированная последовательность чисел от 1 до 100 и одно загаданное число. Мы не будем последовательно перебирать значения от 1 до 100. Вместо этого будем брать число по центру рассматриваемого списка, смотреть, взяли ли мы ***много*** или ***мало***, после чего - отсекать ненужную половину и работать только с верной частью искомого списка.
- При бинарном поиске каждый раз исключается половина чисел.
- Для списка из `n` элементов Бинарник выполняется за `log2 n` шагов, тогда как простой поиск будет выполнен за `n` шагов.
***
***Реализация.***
```python
def binary_search(list, item):
	low = 0 # нижняя граница
	high = len(list) - 1 # верхняя граница
	
	while low <= high: # пока не будет 1 элемент
		mid = (low + high) // 2
		guess = list[mid]
		if guess == item: # нашли
			return mid
		elif guess > item: # перебор
			high = mid - 1
		else: # недобор
			low = mid + 1
	return None
```
